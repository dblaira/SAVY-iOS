# Figma Brief

Figma is the design exploration surface. Xcode is the implementation surface.

## First File

Working name: `SAVY iOS Native`

## First Frames

- Home: SAVY title, leverage principle, latest captures, native surface list.
- Capture sheet: title, meaning, optional follow-up, optional context.
- Entry detail: full meaning, metadata, follow-up controls.
- Native system map: notifications, location, photos, widgets, intents, share extension.

## Visual Direction

- Keep the SAVY red/cream/ink identity, but let iOS controls feel native.
- Do not recreate the website as screens.
- Use real iPhone interaction patterns: sheets, navigation stacks, forms, toggles, pickers, system icons, haptics.
- Adam reaction is the bar. If the wording or layout feels generic, it is wrong.

## Handoff Rule

Every Figma decision should map to a SwiftUI/UIKit implementation. Avoid design-only effects that fight native behavior.
